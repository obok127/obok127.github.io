Processing Rules & Variables Alignment Tool

API-based comparison tool for Processing Rules and variable metadata, built to replace slow manual audits with repeatable configuration-drift analysis.

ProblemManual audits required multiple people to compare country RSIDs, rules, and hundreds of metrics/dimensions; fixes were also hard to re-check consistently.
ScaleRSID lookup, Processing Rule extraction, eVar/prop/event metadata extraction, country-level variable matrix, and drift analysis reports.
BuildFastAPI backend, React/TypeScript UI, Adobe Analytics API clients, OAuth/WSSE auth handling, Excel/CSV export, and field-level diffs for left-only, right-only, and changed-same-title cases.
ImpactReduced an audit flow previously estimated at 3 people over 2–3 weeks into a repeatable workflow executable by one person in under 30 minutes.
EvidenceDemo walkthrough, runnable app structure, generated Excel outputs, and drift report sheets for summary, missing, extra, and changed configurations.

Historical Decision Intelligence

Issue intelligence system for tagging decisions, ownership, timeline reconstruction, and source-grounded answers across Jira, email, Confluence-like documents, and collaboration history.

ProblemTagging issues rarely live in one ticket. Decisions are spread across comments, emails, specs, old QA evidence, and follow-up discussions, so analysts spend time reconstructing context before they can act.
DesignNot a plain document chatbot. The system models issue history as a decision timeline with entities such as country, platform, page, variable, event, owner, status, latest decision, and source evidence.
BuildPeriodic ingestion, entity extraction, issue-thread linking, temporal ranking, decision-state resolution, and answer templates that return latest decision, key history, related owners, and original source links.
GuardrailSeparate source evidence from generated synthesis, expose unresolved conflicts, and keep final operational decisions in a human review loop.

Computer Vision QA

Tagging QA Automation Tool

AI-assisted QA workflow that compares UI/CTA evidence against tagging specifications and turns exhaustive page-by-page manual checks into alert-based review.

ProblemFor each page, reviewers often need to open planning documents, tagging guides, UI references, and QA evidence manually. A single page can take at least half a day of one person’s time when every CTA and expected event must be checked line by line.
BuildUse UI screenshots or design references, tagging guide entries, CTA labels, expected variables/events, and image/text embeddings to match visible components with expected tracking rules.
OutputAlert report for likely missing tags, mismatched CTA naming, wrong variable mapping, duplicate coverage, or low-confidence UI/spec matches, so the reviewer investigates exceptions instead of re-checking every element.
ValueReframes tagging QA from exhaustive manual inspection into AI-assisted triage while keeping final approval with a human reviewer.

Product Attribution QA

SEA to MST Rule Simulation

Implemented marketing-processing-rule simulation for migration QA, designed to prevent attribution drift before migrated campaign traffic is trusted by business reporting.

ProblemDuring SEA to MST migration, marketing-channel processing rules must classify legacy tracking codes and CIDs correctly. If the migration rules are wrong, campaign performance measurement can become unusable or misleading.
BuildExtracted one year of historical tracking-code/CID data, attached SEA migration parameter scenarios, simulated the rule logic, and verified whether each pattern would be classified into the expected marketing channel.
ValidationChecked rule precedence, parameter coverage, expected outputs, mismatch cases, and high-risk tracking-code patterns before the migration was treated as report-ready.
ImpactPrevented business-facing marketing measurement issues before launch. The migration validation relied on this simulation as the primary check, while source code remains internal due to security constraints.
